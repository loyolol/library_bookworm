# 📚 Книжный червь — веб-приложение для учета выдачи книг в библиотеке

**Книжный червь** — это полнофункциональное веб-приложение, разработанное для автоматизации учета выдачи книг в небольших библиотеках, школьных библиотеках, вузовских кафедральных собраниях или личных коллекциях.

## 📋 О проекте

Приложение позволяет пользователям просматривать каталог книг, бронировать доступные издания и управлять своими бронированиями. Администраторы имеют возможность добавлять новые книги, редактировать существующие записи и удалять книги при отсутствии активных бронирований.

Проект разработан в рамках курсовой работы по дисциплине «Проектирование и разработка веб-приложений».

## ✨ Возможности

### Для всех пользователей
- Просмотр каталога книг с обложками
- Детальная информация о каждой книге
- Регистрация и авторизация в системе

### Для авторизованных читателей
- Бронирование доступных книг
- Просмотр списка своих бронирований
- Отмена активных бронирований
- Отслеживание статуса бронирования (активно, возвращено, просрочено, отменено)

### Для администраторов
- Полное управление книжным фондом
- Добавление новых книг с загрузкой обложки (файл или URL)
- Редактирование информации о книгах
- Удаление книг (только при отсутствии активных бронирований)
- Предварительный просмотр обложек при загрузке

## 🛠 Технологии

- **Backend**: PHP (нативный, без фреймворков)
- **База данных**: PostgreSQL
- **Frontend**: HTML5, CSS3, JavaScript (нативный)
- **Работа с БД**: PDO (подготовленные запросы)
- **Безопасность**: хеширование паролей, защита от SQL-инъекций и XSS-атак

## 🗄 База данных

Приложение использует PostgreSQL. Структура базы данных включает три таблицы:

### users (пользователи)
- `user_id` — первичный ключ
- `name` — имя пользователя
- `email` — email (уникальный)
- `password_hash` — хеш пароля
- `role` — роль (user/admin)
- `registration_date` — дата регистрации

### books (книги)
- `book_id` — первичный ключ
- `title` — название
- `author` — автор
- `publication_year` — год издания
- `publisher` — издательство
- `genre` — жанр
- `total_quantity` — общее количество
- `quantity_available` — доступное количество
- `cover_path` — путь к обложке
- `description` — описание

### bookings (бронирования)
- `booking_id` — первичный ключ
- `user_id` — внешний ключ к users
- `book_id` — внешний ключ к books
- `booking_date` — дата бронирования
- `due_date` — срок возврата
- `status` — статус (active/returned/overdue/cancelled)

## 🚀 Установка и запуск

### Требования
- Веб-сервер (Apache / Nginx)
- PHP 7.4 или выше
- PostgreSQL 12 или выше

### Инструкция по установке

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/loyolol/library_bookworm.git
   ```

2. **Настройте базу данных**
   - Создайте базу данных `library_db` в PostgreSQL
   - Выполните SQL-код из файла `library_db.sql` для создания таблиц

3. **Настройте подключение к БД**
   - Откройте файл `config.php`
   - Укажите свои параметры подключения:
   ```php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'library_db');
   define('DB_USER', 'имя_пользователя');
   define('DB_PASS', 'ваш_пароль');
   define('DB_PORT', '5432');
   ```

4. **Настройте права на директорию для загрузок**
   ```bash
   chmod 777 uploads/covers/
   ```

5. **Запустите приложение**
   - Разместите файлы проекта в корневой директории вашего веб-сервера
   - Откройте в браузере `php -S localhost:8000`

## 🔒 Безопасность

- Пароли хранятся в зашифрованном виде (password_hash)
- Защита от SQL-инъекций через подготовленные запросы PDO
- Экранирование вывода для предотвращения XSS-атак
- Проверка прав доступа на сервере
- Транзакции для операций бронирования и отмены
- Валидация загружаемых файлов (только JPG, PNG, GIF)
